\section{Description}

Ce document décrit l'ensemble des tests prévus sur la partie \emph{Core} de l'application.
Il recense les test unitaires et planifie leur déroulement. Il décrit les moyens utilisés pour procéder aux vérifications.

\section{Généralités}

\subsection{Spécificités - Moyens}

Le projet repose sur la production et le traitement d'ondes ramenées à des valeurs numériques.
Les tests unitaires classiques sont complétement inadaptés au test des valeurs produites par les modules.
Aussi le test de ces modules passera par un affichage des valeurs produites sur un \emph{oscilloscope}, avec contrôle visuel de la bonne forme des ondes.
Des tests unitaires classiques seront opérés sur les éléments du \emph{Core} autres que les modules.

\subsection{Réalisation}

Les tests visuels (modules) réalisés feront l'objet de compte-rendus dans un document \emph{tests\_modules.tex}.
Celui-ci précisera :
\begin{itemize}
 \item (en titre de section) la personne et la date de la série de tests,
 \item la liste des tests effectués,
 \item le cas échéant, les erreurs corrigées.
\end{itemize}
\smallskip

Les tests unitaires classiques (éléments du \emph{Core}) feront l'objet de compte-rendus dans un document \emph{tests\_core.tex}.
À chaque fois, tous les tests automatiques seront exécutés.
Celui-ci précisera :
\begin{itemize}
 \item (en titre de section) la personne et la date de la série de tests,
 \item le cas échéant, les erreurs repérées, ainsi que la nature des corrections.
\end{itemize}
